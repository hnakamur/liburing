65 /*
66  * We first setup 4 timeout requests, which require a count value of 1, 1, 2,
67  * UINT_MAX, so the sequence is 1, 2, 4, 2. Before really timeout, this 4
68  * requests will not lead the change of cq_cached_tail, so as sq_dropped.
69  *
70  * And before this patch. The order of this four requests will be req1->req2->
71  * req4->req3. Actually, it should be req1->req2->req3->req4.
72  *
73  * Then, if there is 2 nop req. All timeout requests expect req4 will completed
74  * successful after the patch. And req1/req2 will completed successful with
75  * req3/req4 return -ETIME without this patch!
